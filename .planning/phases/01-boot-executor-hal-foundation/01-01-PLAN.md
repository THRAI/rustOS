  <action>
    Create the project scaffold per BLACKBOX Section 10 layout:

    1. **rust-toolchain.toml**: Pin nightly (use `nightly-2025-06-01` or latest known-good), minimal profile, components: rust-src, llvm-tools, rustfmt, clippy. Target: riscv64gc-unknown-none-elf.

    2. **Root Cargo.toml**: Workspace with members = ["kernel", "hal-common"]. Workspace dependencies for async-task, bitflags, log, buddy_system_allocator (all path deps to vendor/ -- create stubs if vendor dirs don't exist yet, actual vendoring happens when deps are first used).

    3. **.cargo/config.toml**: Set default target for kernel crate to riscv64gc-unknown-none-elf. Set rustflags for kernel: `-C link-arg=-Tkernel/linker/rv64-qemu.ld -C force-frame-pointers=yes`. Set runner for QEMU.

    4. **kernel/Cargo.toml**: `#![no_std]` binary crate. Depends on hal-common (path). Features: `custom_test_frameworks` for Tier 2 later.

    5. **kernel/linker/rv64-qemu.ld**: BASE_ADDRESS = 0x80200000. Sections: .text.entry FIRST (contains _start), .text, .rodata, .data, .bss (with sbss/ebss symbols). Boot stacks at end: 16KB per hart, MAX_CPUS=8 harts = 128KB total. Symbols: boot_stack_bottom, boot_stack_top. ENTRY(_start).

    6. **kernel/src/hal/rv64/boot.S**: Global `_start`:
       - a0 = hartid, a1 = dtb_ptr (passed by OpenSBI)
       - If hartid >= MAX_CPUS (8): park with `wfi` loop
       - Hart 0 only: clear .bss (sbss to ebss)
       - All valid harts: compute sp = boot_stack_top - (hartid * 16384)
       - **Write 0 to sscratch** (critical: prevents corrupted sp on first trap)
       - Call `rust_main(hartid, dtb_ptr)` -- extern "C" fn in main.rs
       - If rust_main returns: `wfi` loop (should never happen)

    7. **kernel/src/main.rs**: `#![no_std]`, `#![no_main]`, panic handler that loops. `#[no_mangle] pub extern "C" fn rust_main(hartid: usize, dtb_ptr: usize)` -- for now just calls UART init and prints "hello world", then loops with `wfi`.

    8. **kernel/src/hal/mod.rs** and **kernel/src/hal/rv64/mod.rs**: Module structure. rv64/mod.rs re-exports boot, uart, irq submodules.

    9. **hal-common/Cargo.toml**: Pure `#![no_std]` library. No arch-specific deps. Dev-dependencies: (empty for now, loom added in Plan 03).

    10. **hal-common/src/lib.rs**: Stub with `#![no_std]`, re-exports addr and errno modules.

    11. **Makefile**: Targets:
        - `kernel-rv64`: `cargo build --release -p kernel --target riscv64gc-unknown-none-elf`, then `rust-objcopy -O binary` to produce kernel-rv64.bin
        - `run-rv64`: QEMU command from BLACKBOX Section 10 (qemu-system-riscv64 -machine virt -nographic -bios firmware/opensbi-rv64.bin -kernel kernel-rv64.bin)
        - `test`: `cargo test --lib -p hal-common` (host tests)
        - `clean`: cargo clean + rm binaries

    Verify boot.S uses `.section .text.entry` so linker places _start first. Verify sscratch = 0 write happens before any trap can fire.
  </action>

<task type="auto">
  <name>Task 2: UART driver + kprintln! + IRQ ops + hal-common types (PhysAddr, VirtAddr, Errno, IrqSafeSpinLock)</name>
  <files>
    kernel/src/hal/rv64/uart.rs
    kernel/src/hal/rv64/irq.rs
    hal-common/src/addr.rs
    hal-common/src/errno.rs
    hal-common/src/irq_lock.rs

  <action>
    1. **kernel/src/hal/rv64/uart.rs**: UART 16550A driver for QEMU virt platform (MMIO at 0x1000_0000). Implement:
       - `pub fn init()` -- configure baud rate divisor, enable FIFO, 8N1
       - `pub fn putchar(c: u8)` -- write byte, spin-wait on THR empty
       - `pub fn getchar() -> Option<u8>` -- non-blocking read from RBR
       - Implement `core::fmt::Write` for a `Uart` struct so `write!` works
       - Use a static `SpinLock<Uart>` (or IrqSafeSpinLock once available) for safe concurrent access

    2. **kprintln! macro**: In kernel/src/ (e.g., kernel/src/console.rs or inline in main.rs):
       - `kprint!` and `kprintln!` macros that call UART write
       - Use `core::fmt::Write` trait
       - Must work from any context (boot, trap handler, executor)

    3. **kernel/src/hal/rv64/irq.rs**: IRQ control for rv64:
       - `pub fn enable()` -- set sstatus.SIE bit via `csrsi`
       - `pub fn disable()` -- clear sstatus.SIE bit via `csrci`
       - `pub fn disable_and_save() -> usize` -- read sstatus, clear SIE, return old value
       - `pub fn restore(saved: usize)` -- restore SIE bit from saved value
       - `pub fn is_enabled() -> bool` -- check sstatus.SIE

    4. **hal-common/src/addr.rs**: Newtype wrappers:
       - `PhysAddr(usize)` with `as_usize()`, `add(offset)`, `page_offset()`, `page_align_down()`, `page_align_up()`
       - `VirtAddr(usize)` with same methods
       - `PageSize` enum or const (4096)
       - Implement `Debug`, `Copy`, `Clone`, `PartialEq`, `Eq`, `PartialOrd`, `Ord` for both

    5. **hal-common/src/errno.rs**: `Errno` enum with common kernel error codes: `ENOMEM`, `EFAULT`, `EINVAL`, `ENOENT`, `EBADF`, `EAGAIN`, `EPERM`, `ENOSYS`. Derive Debug, Copy, Clone, PartialEq.

    6. **hal-common/src/irq_lock.rs**: `IrqSafeSpinLock<T>`:
       - Inner: `spin::Mutex<T>` (or hand-rolled SpinLock using AtomicBool)
       - `lock(&self) -> IrqSafeGuard<T>`: calls arch-specific `disable_and_save()`, then acquires inner lock
       - `IrqSafeGuard<T>`: on Drop, releases inner lock then calls `restore(saved)`
       - **Critical**: The IRQ disable/restore must be arch-abstracted. Use a trait or cfg-gated function pointer. For host tests (x86_64), `disable_and_save` is a no-op that returns 0, `restore` is a no-op. For rv64, it uses the real CSR ops.
       - This ensures `cargo test --lib -p hal-common` works on the host

    7. Update **hal-common/src/lib.rs** to re-export all modules.

    The IrqSafeSpinLock is the ONLY lock type allowed for run queues and any data touched by IRQ handlers. This is a locked decision from CONTEXT.md.
  </action>

  </verify>
