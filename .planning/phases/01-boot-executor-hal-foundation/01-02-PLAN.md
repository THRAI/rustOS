  <action>
    1. **hal-common/src/trap_frame.rs**: Define `TrapFrame` struct (MI):
       - 32 `usize` fields for x0-x31 (x0 always 0 but included for indexing)
       - `sstatus: usize`, `sepc: usize`, `scause: usize`, `stval: usize`
       - `kernel_tp: usize` (saved/restored so per-CPU data survives trap)
       - MI accessor methods:
         - `pc() -> usize` (returns sepc)
         - `set_pc(val: usize)`
         - `arg(n: usize) -> usize` (a0=x10..a7=x17, maps n to register index)
         - `set_arg(n: usize, val: usize)`
         - `set_ret_val(val: usize)` (sets a0)
         - `sp() -> usize` (x2)
         - `ra() -> usize` (x1)
         - `scause() -> usize`
         - `stval() -> usize`
       - `#[repr(C)]` for asm compatibility. Derive Debug.
       - Add to hal-common/src/lib.rs re-exports.

    2. **kernel/src/hal/rv64/trap.S**: `__kernel_trap` (global, naked):
       - Phase 1 only exercises kernel traps (no userspace yet)
       - On entry: sp is valid kernel stack (boot stack), sscratch == 0 (kernel mode)
       - Subtract sizeof(TrapFrame) from sp (~280 bytes, 35 * 8)
       - Save all 32 GPRs to TrapFrame at [sp + offset] using `sd` instructions
       - Save CSRs: `csrr t0, sstatus; sd t0, [sp+sstatus_off]` etc for sepc, scause, stval
       - Save kernel tp: `sd tp, [sp+kernel_tp_off]`
       - Set a0 = sp (pointer to TrapFrame)
       - Call `kernel_trap_handler` (extern "C" fn in trap.rs)
       - On return: restore all CSRs from TrapFrame, restore all 32 GPRs, add sizeof(TrapFrame) to sp
       - `sret`
       - **Also define `__user_trap` stub** (for Phase 1: just redirects to __kernel_trap since no userspace. Will be expanded in Phase 3). The stub exists so stvec switching code compiles.

    3. **kernel/src/hal/rv64/trap.rs**: Trap setup and dispatch:
       - `pub fn init()`: Write stvec to point to `__kernel_trap` (Direct mode, not vectored). Enable timer interrupt in sie (set STIE bit).
       - `#[no_mangle] pub extern "C" fn kernel_trap_handler(frame: &mut TrapFrame)`:
         - Read scause from frame
         - Classify: bit 63 set = interrupt, bits 62:0 = cause code
         - Interrupts: cause 5 = S-mode timer → call `timer::handle_timer_irq()`; cause 9 = S-mode external → call `handle_external_irq()` (stub for now); cause 1 = S-mode software (IPI) → call `handle_ipi()` (stub)
         - Exceptions: cause 8 = ecall from U-mode → stub (Phase 3); cause 12/13/15 = page faults → stub; others → panic with cause info via kprintln!
       - `pub fn set_kernel_trap_entry()`: write stvec = __kernel_trap
       - `pub fn set_user_trap_entry()`: write stvec = __user_trap (stub for Phase 3)

    4. **kernel/src/trap.rs** (MI trap module): Re-export arch-specific trap init. This is the MI entry point that other kernel code calls.

    5. Wire up: In rust_main (from Plan 01), after UART init, call `trap::init()` to set stvec and enable timer interrupts. Then enable global interrupts via `irq::enable()`.

    Per CONTEXT.md decisions:
    - Single trap vector + Rust dispatch (not vectored mode)
    - sscratch == 0 means kernel mode (already set in boot.S from Plan 01)
    - TrapFrame includes kernel_tp
    - Two separate asm vectors (__user_trap and __kernel_trap), stvec rewritten on transitions
  </action>

TrapFrame size matches asm offsets (add a compile-time assert: `const_assert!(core::mem::size_of::<TrapFrame>() == 35 * 8)` or similar).

  <action>
    1. **kernel/src/hal/rv64/timer.rs**:
       - `pub fn init()`: Read timebase frequency from FDT (or hardcode 10MHz for QEMU virt). Set first timer interrupt.
       - `pub fn set_oneshot(ticks: u64)`: Program next timer IRQ. On rv64 QEMU, use SBI set_timer ecall (legacy extension 0x00, or Timer extension EID 0x54494D45 FID 0). Writes `time + ticks` as the compare value.
       - `pub fn handle_timer_irq()`: Called from trap dispatch on S-timer interrupt. Clear timer pending (set next timer via set_oneshot). Print tick count via kprintln! (temporary, replaced by executor integration in Plan 04).
       - `pub fn read_time() -> u64`: Read `time` CSR via `csrr`.
       - `const TICK_HZ: u64 = 10_000_000` (QEMU virt timebase). `const TICK_MS: u64 = TICK_HZ / 1000` (ticks per millisecond).
       - Timer granularity: 10ms per CONTEXT.md. So default oneshot = `TICK_MS * 10`.

    2. **kernel/src/hal/rv64/tlb.rs**: TLB flush operations:
       - `pub fn flush_addr(vaddr: usize)`: `sfence.vma vaddr, zero` (flush single address, all ASIDs)
       - `pub fn flush_asid(asid: usize)`: `sfence.vma zero, asid` (flush all entries for ASID)
       - `pub fn flush_all()`: `sfence.vma zero, zero` (flush everything)
       - All implemented via inline `asm!` with appropriate clobbers.

    3. **kernel/src/hal/rv64/cache.rs**: Cache operations for DMA correctness:
       - `pub fn dcache_wb(addr: usize, len: usize)`: On rv64, `fence w, w` is typically sufficient (QEMU virt has coherent DMA). Implement as `fence` instruction. Add TODO comment noting real hardware may need CBO instructions.
       - `pub fn dcache_inv(addr: usize, len: usize)`: Same -- `fence r, r` or full `fence`. Note: rv64 Zicbom extension provides `cbo.inval` but QEMU virt doesn't need it.
       - `pub fn fence_io()`: `fence iorw, iorw` for MMIO ordering.

    4. Update **kernel/src/hal/rv64/mod.rs** to export timer, tlb, cache modules.

    5. Wire timer: In rust_main, after trap::init(), call timer::init() to arm the first timer interrupt. The timer IRQ should now fire periodically (every 10ms) and be handled by handle_timer_irq.
  </action>
