  <action>
    1. **hal-common/Cargo.toml**: Add dev-dependencies: `loom = "0.7"` (or latest). Add `[features] loom = ["dep:loom"]`. The loom feature gates loom-specific imports.

    2. **hal-common/src/run_queue.rs**: Generic `RunQueue<R>`:
       - Inner: `IrqSafeSpinLock<VecDeque<R>>` (uses IrqSafeSpinLock from Plan 01)
       - `pub fn new() -> Self`
       - `pub fn push(&self, item: R)`: Lock, push_back
       - `pub fn pop(&self) -> Option<R>`: Lock, pop_front
       - `pub fn is_empty(&self) -> bool`: Lock, check
       - `pub fn len(&self) -> usize`: Lock, len
       - The generic R will be `async_task::Runnable` in the kernel, but for host tests it can be any type.

    3. **hal-common/src/timer_wheel.rs**: Single-layer timer wheel:
       - `const SLOTS: usize = 256` (per CONTEXT.md discretion: 256 slots)
       - `const TICK_MS: u64 = 10` (10ms granularity per CONTEXT.md)
       - Each slot: `Vec<TimerEntry>` (or small-vec)
       - `TimerEntry { id: u64, deadline_tick: u64, waker: Option<Waker>, cancelled: bool }`
       - `pub fn new() -> Self`
       - `pub fn insert(&mut self, delay_ms: u64, waker: Waker) -> u64`: Compute slot = (current_tick + delay_ticks) % SLOTS. Push entry. Return timer ID. O(1).
       - `pub fn cancel(&mut self, id: u64)`: Find entry by ID, set cancelled = true. O(1) amortized (skip on expire).
       - `pub fn tick(&mut self, current_tick: u64)`: Advance to current_tick. For each expired slot, wake non-cancelled entries, remove all. O(expired entries).
       - `pub fn advance(&mut self)`: Increment internal tick counter by 1, process current slot.
       - Timer wheel does NOT hold the IrqSafeSpinLock itself -- the PerCpu struct wraps it in one. This keeps the timer wheel pure and testable.

    4. **Host tests** (in hal-common/src/run_queue.rs `#[cfg(test)]` and hal-common/src/timer_wheel.rs `#[cfg(test)]`):
       - RunQueue: push 10 items, pop in FIFO order. Push from multiple threads (std::thread), verify all items popped. Empty queue returns None.
       - TimerWheel: Insert timer at 50ms, advance 4 ticks (40ms) → not expired. Advance 1 more (50ms) → waker called. Cancel timer before expiry → waker NOT called. Insert multiple timers at same slot → all wake. Insert timer at 0 delay → expires on next tick.
       - IrqSafeSpinLock: Lock from two threads, verify mutual exclusion (counter increment test).

    5. **Loom tests** (in hal-common/src/irq_lock.rs or separate test file, gated by `#[cfg(loom)]`):
       - Two threads: thread A locks IrqSafeSpinLock, increments counter, unlocks. Thread B does same. Verify final count == 2.
       - Simulate IRQ-reentrance scenario: thread acquires lock, "IRQ" (another loom thread) tries to acquire same lock → verify it spins (doesn't deadlock because loom explores all interleavings). Note: with IrqSafeSpinLock, IRQs are disabled before lock, so the "IRQ" thread represents a different CPU, not same-CPU reentrance.

    6. Update **hal-common/src/lib.rs** to export run_queue and timer_wheel modules.
  </action>

  <action>
    1. **kernel/src/executor/per_cpu.rs**: `PerCpu` struct:
       - `run_queue: RunQueue<async_task::Runnable>`
       - `timer_wheel: IrqSafeSpinLock<TimerWheel>`
       - `hartid: usize` (physical hart ID)
       - `cpu_id: usize` (logical CPU ID, 0-based)
       - Global: `static PER_CPU_DATA: [PerCpu; MAX_CPUS]` (initialized at boot)
       - `pub fn current() -> &'static PerCpu`: Read tp register, interpret as pointer to PerCpu. For Phase 1, tp is set in rust_main after FDT parse (per CONTEXT.md: two-stage tp setup).
       - `pub fn get(cpu_id: usize) -> &'static PerCpu`: Index into PER_CPU_DATA array.

    2. **kernel/src/executor/schedule.rs**: async-task integration:
       - `pub fn spawn_kernel_task<F>(future: F, target_cpu: usize) -> async_task::Task<F::Output>` where F: Future + Send + 'static:
         - Create schedule_fn closure: `move |runnable| { PerCpu::get(target_cpu).run_queue.push(runnable); /* IPI if target_cpu != current_cpu */ }`
         - Call `async_task::spawn(future, schedule_fn)` → returns (Runnable, Task)
         - Schedule the initial Runnable: `runnable.schedule()`
         - Return Task (caller can `.detach()` it)
       - `pub async fn sleep(ms: u64)`: Returns a future that:
         - On first poll: inserts into current CPU's timer_wheel with waker from cx
         - On subsequent polls: checks if timer expired, returns Ready if so
         - Use `core::future::poll_fn` or a custom SleepFuture struct
       - `pub async fn yield_now()`: Returns Pending once (reschedules), then Ready. Used for cooperative preemption.

    3. **kernel/src/executor/mod.rs**: Executor loop:
       - `pub fn executor_loop() -> !`:
         - Loop forever:
           - Pop from current CPU's run_queue
           - If Some(runnable): `runnable.run()` (polls the future once)
           - If None (queue empty): idle with `wfi` instruction
           - **Critical idle sequence per CONTEXT.md:** unlock queue (IRQs restored) → check empty → if empty: `csrsi sstatus, SIE; wfi; csrci sstatus, SIE` → re-check
         - This is the only place futures get polled

    4. **Wire up in kernel/src/main.rs**:
       - In rust_main, after trap and timer init:
         - Initialize PerCpu for hart 0 (set tp register via inline asm)
         - Spawn a test kernel task: `spawn_kernel_task(async { kprintln!("hello from async future!"); }, 0).detach()`
         - Enter `executor_loop()` (never returns)
       - The test task proves the full path: spawn → schedule_fn → push to run queue → executor pops → runs → prints

    5. **Module structure**: kernel/src/executor/mod.rs re-exports per_cpu, schedule submodules.

    Per CONTEXT.md locked decisions:
    - Single FIFO VecDeque per CPU (no priority levels)
    - IrqSafeSpinLock enforced (from hal-common)
    - async-task crate for spawn/schedule
    - Idle loop uses wfi with SIE enable/disable sequence
  </action>

  </verify>
