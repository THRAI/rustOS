  <action>
    1. **kernel/src/hal/rv64/sbi.rs**: SBI ecall wrappers:
       - `fn sbi_call(eid: usize, fid: usize, args: [usize; 3]) -> SbiRet`: Inline asm ecall with a0-a2 = args, a6 = fid, a7 = eid. Returns SbiRet { error: a0, value: a1 }.
       - `pub fn set_timer(stime: u64)`: EID 0x54494D45, FID 0. (Or legacy EID 0x00 if simpler.)
       - `pub fn send_ipi(hart_mask: usize, hart_mask_base: usize)`: EID 0x735049, FID 0 (sPI extension). Sends S-mode software interrupt to target harts.
       - `pub fn hart_start(hartid: usize, start_addr: usize, opaque: usize)`: EID 0x48534D, FID 0 (HSM extension). Starts a stopped hart at start_addr with a0=opaque.
       - `pub fn hart_status(hartid: usize) -> usize`: EID 0x48534D, FID 2. Query hart state.

    2. **kernel/src/hal/rv64/fdt.rs**: Minimal FDT parsing:
       - `pub fn parse_cpus(dtb_ptr: usize) -> (usize, [usize; MAX_CPUS])`: Parse /cpus node from FDT. Return (num_cpus, hartid_array). Use the `fdt` crate from vendor/ if available, otherwise hand-parse the FDT header + /cpus node (it's a simple structure).
       - Build hartid → logical_cpu_id mapping. Harts beyond MAX_CPUS are ignored.
       - Store mapping in a global: `static HART_TO_CPU: [Option<usize>; MAX_HARTS]`.

    3. **kernel/src/hal/rv64/ipi.rs**:
       - `pub fn send_ipi(target_cpu: usize)`: Look up physical hartid from logical cpu_id, call `sbi::send_ipi` with appropriate hart_mask.
       - `pub fn handle_ipi()`: Called from trap dispatch on S-software interrupt (scause cause 1). Clear sip.SSIP bit. The IPI's purpose is just to break `wfi` -- the actual work (popping from run queue) happens in the executor loop.

    4. **kernel/src/hal/rv64/smp.rs**:
       - `pub fn boot_secondary_harts(num_cpus: usize, hartids: &[usize])`: For each hart (except boot hart 0):
         - Call `sbi::hart_start(hartid, secondary_entry as usize, logical_cpu_id)`
         - `secondary_entry` is an asm stub (or Rust fn) that: sets sp from boot stack (same formula as boot.S), sets sscratch = 0, calls `secondary_rust_main(cpu_id)`
       - `secondary_rust_main(cpu_id: usize)`: Initialize PerCpu for this CPU (set tp register), init trap vectors, init timer, enter `executor_loop()`.

    5. **Update kernel/src/executor/schedule.rs**: In `schedule_fn` closure, after pushing Runnable to target CPU's run queue, if `target_cpu != current_cpu`, call `ipi::send_ipi(target_cpu)` to wake the remote CPU from wfi.

    6. **Update kernel/src/executor/per_cpu.rs**: Add `pub fn init(cpu_id: usize, hartid: usize)` method that sets fields and writes tp register.

    7. **Update kernel/src/main.rs** (rust_main for hart 0):
       - Parse FDT to discover CPUs: `let (num_cpus, hartids) = fdt::parse_cpus(dtb_ptr)`
       - Initialize PerCpu for hart 0
       - Set tp register: `asm!("mv tp, {}", in(reg) &PER_CPU_DATA[0])` (per CONTEXT.md two-stage tp setup)
       - Boot secondary harts: `smp::boot_secondary_harts(num_cpus, &hartids)`
       - Spawn test tasks on different CPUs to verify cross-CPU wake
       - Enter executor_loop()

    Per CONTEXT.md locked decisions:
    - SBI HSM for hart_start
    - tp register → PerCpu struct
    - Two-stage tp setup (boot.S uses raw hartid, rust_main sets tp to PerCpu pointer)
    - FDT discovery + MAX_CPUS=8 cap
    - IPI via SBI send_ipi
  </action>

  <action>
    1. **Timer wheel integration** (kernel/src/hal/rv64/timer.rs + kernel/src/executor/mod.rs):
       - In `handle_timer_irq()`: After rearming the timer via set_oneshot, call `PerCpu::current().timer_wheel.lock().advance()` to tick the timer wheel. This wakes any expired sleep futures (their wakers fire, which calls schedule_fn, which pushes Runnables to run queues).
       - This connects the hardware timer IRQ → timer wheel → sleep future wake path.

    2. **Preemption** (kernel/src/hal/rv64/trap.rs):
       - In timer IRQ handler: Set a per-CPU `needs_reschedule` flag (AtomicBool on PerCpu).
       - The `yield_now()` future (from Plan 03) checks this flag. When the executor polls a future and it hits a `.await` point, if `needs_reschedule` is set, yield_now returns Pending (causing the executor to move to the next task).
       - For Phase 1 (no userspace), preemption at trap boundary means: timer IRQ fires during executor idle or during a kernel future's execution → trap handler sets flag → future yields at next .await → executor polls next task.
       - This is the chronix pattern per CONTEXT.md.

    3. **Integration test: sleep future**:
       - In rust_main, spawn a kernel task: `spawn_kernel_task(async { sleep(100).await; kprintln!("woke after 100ms!"); }, 0).detach()`
       - Verify it prints after ~100ms (10 timer ticks at 10ms each).

    4. **Integration test: cross-CPU wake**:
       - Spawn task A on CPU 0 that sleeps 50ms then spawns task B on CPU 1.
       - Task B prints "hello from CPU 1".
       - Verify both messages appear in QEMU output.

    5. **Debug infrastructure** (BOOT-09):
       - kprintln! already exists from Plan 01.
       - **GDB stub support**: Add `debug-rv64` Makefile target: `qemu-system-riscv64 ... -s -S` (GDB server on port 1234, halt on start). Add `gdbserver-rv64` target without -S (don't halt).
       - **QEMU trace flags**: Add `QEMU_TRACE` variable to Makefile. `make run-rv64 QEMU_TRACE="-d int,cpu_reset"` appends trace flags.
       - **Register clobber test**: In rust_main (or as a separate test function), write known values to all caller-saved registers (t0-t6, a0-a7), trigger a trap (e.g., `ecall` or wait for timer IRQ), then verify registers are restored. This tests that __kernel_trap saves/restores correctly. Print PASS/FAIL via kprintln!.

       - `debug-rv64`: QEMU with `-s -S` flags.
       - `test-all`: Run `test` (host) + `qemu-test-rv64`.
