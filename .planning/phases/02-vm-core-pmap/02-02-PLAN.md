**1. Create os/src/mm/allocator/buddy.rs:**

Implement a buddy system allocator for physical page frames:
- `BuddyAllocator` struct with free lists for orders 0..MAX_ORDER (MAX_ORDER=11, up to 2^11 = 2048 contiguous pages = 8MB blocks)
- Each order has a linked list of free blocks (use a simple `Vec<PhysPageNum>` per order for now — the bitmap allocator already uses Vec)
- `init(range: Range<PhysPageNum>)` — add initial memory range, splitting into appropriate buddy blocks
- `alloc(order: usize) -> Option<PhysPageNum>` — find smallest available block >= requested order, split if needed
- `free(ppn: PhysPageNum, order: usize)` — return block, coalesce with buddy if buddy is also free
- `available_pages() -> usize` — total free pages across all orders
- Buddy address calculation: `buddy_ppn = ppn ^ (1 << order)`
- Use `IrqSafeSpinLock` (from existing codebase pattern) to protect the global buddy state
- Track total/free page counts for diagnostics

**2. Create os/src/mm/allocator/magazine.rs:**

Per-CPU magazine layer for single-frame (order-0) fast path:
- `Magazine` struct: fixed-size array of `Option<PhysPageNum>`, capacity 32 frames
- `MagazineCache` struct: per-CPU, holds one active magazine
- `pop() -> Option<PhysPageNum>` — take from magazine (no lock needed, per-CPU)
- `push(ppn: PhysPageNum) -> Result<(), PhysPageNum>` — return to magazine, Err if full
- `refill(&mut self, buddy: &BuddyAllocator, count: usize)` — batch-alloc from buddy to fill magazine
- `drain(&mut self, buddy: &BuddyAllocator)` — return all magazine frames to buddy
- Access via PerCpu (from Phase 1 infrastructure) — each CPU has its own MagazineCache

**3. Memory poisoning (debug builds):**

In frame_allocator.rs or a new debug module:
- `#[cfg(debug_assertions)] const POISON_PATTERN: u64 = 0xDEAD_BEEF_DEAD_BEEF;`
- `poison_frame(ppn: PhysPageNum)` — fill 4096 bytes with POISON_PATTERN on free
- `check_poison(ppn: PhysPageNum)` — assert all bytes match POISON_PATTERN on alloc (before zeroing)
- Trap stack canary: `const STACK_CANARY: u64 = 0xCAFE_BABE_DEAD_C0DE;` — write at bottom of trap stack, check on trap entry
- Wrap in `#[cfg(debug_assertions)]` so release builds have zero overhead

**4. Refactor os/src/mm/allocator/frame_allocator.rs and mod.rs:**

Keep the existing `BitMapFrameAllocator` and `FrameAllocator` struct that implements `FrameAllocatorHal` — the HAL trait is used by the page table code. Add the new dual API alongside:

- `pub fn frame_alloc_sync() -> Option<PhysPageNum>` — synchronous, never yields:
  1. Try per-CPU magazine pop
  2. Try global buddy alloc(0)
  3. Try emergency_reclaim_sync()
  4. Return None (ENOMEM)

- `pub async fn frame_alloc() -> Option<PhysPageNum>` — async-capable:
  1. Try per-CPU magazine pop
  2. Try global buddy alloc(0)
  3. Wake page daemon (stub for now — Phase 5), yield
  4. Try emergency_reclaim_sync()
  5. Return None

- `pub fn frame_free(ppn: PhysPageNum)` — free single frame:
  1. Debug: poison_frame(ppn)
  2. Try per-CPU magazine push
  3. If magazine full: drain half to buddy, then push

- `pub fn emergency_reclaim_sync() -> Option<PhysPageNum>` — stub for now:
  Returns None. Will scan inactive queue in Phase 5 when page daemon exists.
  Add a comment: `// TODO(Phase 5): scan inactive queue for clean cached pages`

- `pub fn frame_alloc_contiguous(order: usize) -> Option<PhysPageNum>` — for multi-page allocs, goes directly to buddy

Wire the buddy allocator initialization into the existing `init_frame_allocator()` function. The buddy allocator should be initialized with the same physical memory range the bitmap allocator currently uses. Keep the bitmap allocator code but gate it behind a feature flag or cfg so we can fall back if needed.

Export the new functions from mod.rs.
