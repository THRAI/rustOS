**1. os/src/mm/pmap/walk.rs — Generic page table walk:**

```rust
use hal::addr::{PhysPageNum, VirtAddr};
```

Define a `PtEntry` trait that abstracts over arch-specific PTE formats:
- `fn is_valid(&self) -> bool`
- `fn is_leaf(&self) -> bool`
- `fn ppn(&self) -> PhysPageNum`
- `fn set(&mut self, ppn: PhysPageNum, flags: PteFlags)`
- `fn flags(&self) -> PteFlags`
- `fn clear(&mut self)`

Define `PteFlags` bitflags: Valid, Read, Write, Execute, User, Global, Accessed, Dirty, Cow (software bit).

Implement the shared walk function:
```rust
pub fn walk_page_table<const LEVELS: usize>(
    root_ppn: PhysPageNum,
    va: VirtAddr,
    alloc_missing: bool,
    allocator: &dyn Fn() -> Option<PhysPageNum>,
) -> Result<(*mut PteRaw, usize), WalkError>
```

Where:
- `LEVELS` is 3 for Sv39, 4 for LA64
- Extracts VPN indices from va using `(va >> (12 + level * 9)) & 0x1FF` for each level
- At each level: read PTE from table, if not valid and alloc_missing then allocate a new page table page via allocator callback, set PTE to point to it
- If PTE is a leaf (superpage), return early with the level
- Returns pointer to the leaf PTE and the level at which it was found
- `WalkError` enum: NotMapped, AllocFailed

The walk operates on raw physical addresses converted to virtual via the kernel's direct-map (existing `Constant::KERNEL_ADDR_SPACE` offset). Use the same phys-to-virt pattern the existing page table code uses.

Only arch-specific parts: how to extract VPN index at each level, and how to determine if a PTE is a leaf. For Sv39: `vpn_index(va, level) = (va >> (12 + level * 9)) & 0x1FF`, leaf = any of RWX set.

**2. os/src/mm/pmap/mod.rs — Pmap struct and public API:**

```rust
pub mod walk;
#[cfg(target_arch = "riscv64")]
mod rv64;
#[cfg(target_arch = "riscv64")]
pub use rv64::*;
```

Define the `Pmap` struct (arch-specific inner in rv64.rs):
- `root: PhysPageNum` — root page table physical page
- `asid: u16` — address space ID (assigned later by ASID allocator, 0 for now)

Public API functions (methods on Pmap):
- `pmap_enter(&mut self, va: VirtAddr, ppn: PhysPageNum, prot: PteFlags) -> Result<(), PmapError>` — walk to leaf, create intermediate tables if needed, set PTE with given flags + Valid + Accessed. If mapping already exists, update it.
- `pmap_remove(&mut self, va: VirtAddr) -> Result<PteFlags, PmapError>` — walk to leaf, clear PTE, return old flags. Does NOT free intermediate page table pages (lazy cleanup).
- `pmap_protect(&mut self, va: VirtAddr, new_prot: PteFlags) -> Result<(), PmapError>` — walk to leaf, update permission bits without changing PPN. Used for COW (remove Write) and mprotect.
- `pmap_extract(&self, va: VirtAddr) -> Option<(PhysPageNum, PteFlags)>` — walk without allocating, return PPN + flags if mapped.
- `pmap_zero_page(ppn: PhysPageNum)` — zero a physical page (for anonymous fault)
- `pmap_copy_page(src: PhysPageNum, dst: PhysPageNum)` — copy 4096 bytes between physical pages (for COW)
- `pmap_is_referenced(&self, va: VirtAddr) -> bool` — check Accessed bit
- `pmap_clear_referenced(&mut self, va: VirtAddr)` — clear Accessed bit (for future page daemon)
- `pmap_is_modified(&self, va: VirtAddr) -> bool` — check Dirty bit
- `pmap_clear_modified(&mut self, va: VirtAddr)` — clear Dirty bit (for future writeback daemon)
- `new(asid: u16) -> Self` — allocate root page table page
- `activate(&self)` — write satp register with this pmap's root + asid

`PmapError` enum: NotMapped, AllocFailed, AlreadyMapped.

  <name>Task 2: RV64 Sv39 pmap implementation</name>

Create os/src/mm/pmap/rv64.rs with the RV64-specific Pmap implementation:

1. Define `Sv39Pte` struct wrapping a `u64` that implements the PtEntry trait from walk.rs:
   - Bit layout per RISC-V Sv39 spec: V(0), R(1), W(2), X(3), U(4), G(5), A(6), D(7), RSW(8-9), PPN(10-53)
   - Use RSW bit 8 for COW software flag
   - `is_valid()` = bit 0
   - `is_leaf()` = any of R|W|X set (bits 1-3)
   - `ppn()` = bits 10..54 >> 10, as PhysPageNum
   - `set(ppn, flags)` = encode PPN into bits 10..54, set flag bits

2. Implement `Pmap` struct for rv64:
   ```rust
   pub struct Pmap {
       root: PhysPageNum,
       asid: u16,
   }
   ```

3. Implement all pmap methods using `walk_page_table::<3>()`:
   - `pmap_enter`: walk with alloc_missing=true, set PTE. Use frame_alloc_sync() as the allocator callback for intermediate page table pages (page table pages are always allocated synchronously).
   - `pmap_remove`: walk with alloc_missing=false, clear PTE.
   - `pmap_protect`: walk with alloc_missing=false, modify flags keeping PPN.
   - `pmap_extract`: walk with alloc_missing=false, read PTE.

4. `activate()`: write `satp` CSR with mode=8 (Sv39), ASID in bits 44-59, PPN of root in bits 0-43. Use `riscv::register::satp::set()` or inline asm `csrw satp, val` followed by `sfence.vma`.

5. `pmap_zero_page` and `pmap_copy_page`: convert PhysPageNum to kernel virtual address (add KERNEL_ADDR_SPACE offset), then use `core::ptr::write_bytes` / `core::ptr::copy_nonoverlapping`.

6. Referenced/Modified bit accessors: walk to PTE, read/clear A and D bits.

The allocator callback for walk should use the existing `FrameAllocator` (which implements FrameAllocatorHal) for allocating intermediate page table pages, since this is the established pattern. When the new buddy allocator from Plan 02 is ready, it will be wired in — but for now use the existing allocator to avoid a hard dependency.
