**1. Create os/src/mm/pmap/asid.rs — Generation-based ASID allocator:**

```rust
struct AsidAllocator {
    next_asid: u16,
    generation: u64,
    max_asid: u16,  // 65535 for RV64 Sv39 (16-bit ASID field in satp)
    lock: IrqSafeSpinLock<()>,  // rollover lock
}
```

- `ASID_BITS` const: 16 for RV64 (from satp encoding)
- `MAX_ASID`: `(1 << ASID_BITS) - 1` = 65535
- Global static: `static ASID_ALLOCATOR: IrqSafeSpinLock<AsidAllocatorInner>`

`AsidAllocatorInner`:
- `next_asid: u16` — next ASID to hand out
- `generation: u64` — incremented on rollover

`AsidEntry` returned to caller:
- `asid: u16`
- `generation: u64`

`alloc_asid() -> AsidEntry`:
1. Lock the allocator
2. If `next_asid < MAX_ASID`: assign, increment, return
3. Else (rollover):
   - Increment generation
   - Reset next_asid to 1 (ASID 0 reserved for kernel)
   - **Synchronous IPI barrier**: send IPI to ALL other CPUs requesting TLB flush_all
   - Spin-wait until all CPUs have acked (per-CPU AtomicBool ack flags)
   - Also flush local TLB
   - Then assign ASID 1, increment, return

Double-check pattern: after acquiring lock, re-check if another CPU already rolled over (generation changed) — if so, just allocate normally without another rollover.

`check_asid(entry: &AsidEntry) -> bool`:
- Returns true if entry.generation matches current generation (ASID still valid)
- If false, caller must call alloc_asid() to get a fresh one

**2. Create os/src/mm/pmap/shootdown.rs — TLB shootdown via IPI:**

```rust
pub fn pmap_shootdown(pmap: &Pmap, va_start: VirtAddr, va_end: VirtAddr)
```

Implementation:
1. Determine which CPUs are running this pmap's ASID (for now, broadcast to all — optimization later)
2. Set per-CPU `shootdown_request` with va range + requesting CPU ID
3. Send IPI to all other CPUs via SBI send_ipi (reuse Phase 1 IPI infrastructure)
4. Each CPU's IPI handler checks shootdown_request:
   - If va range specified: `sfence.vma va, asid` for each page in range
   - If full flush requested: `sfence.vma zero, asid`
   - Set per-CPU ack flag
5. Initiator also flushes local TLB for the range
6. Initiator spins until all target CPUs have acked
7. Clear shootdown state

Per-CPU shootdown state:
```rust
struct ShootdownState {
    pending: AtomicBool,
    va_start: VirtAddr,
    va_end: VirtAddr,
    asid: u16,
    ack: AtomicBool,
}
```

Store in PerCpu struct (extend from Phase 1) or as a separate static array indexed by cpu_id.

**3. Wire pmap_remove and pmap_protect to call shootdown:**

Update pmap_remove and pmap_protect in the Pmap impl to call `pmap_shootdown(self, va, va + PAGE_SIZE)` after modifying the PTE. This ensures MI code above pmap never needs to think about TLB coherence.

Add `pub mod asid;` and `pub mod shootdown;` to os/src/mm/pmap/mod.rs.

Create os/src/mm/vm/fault.rs — sync_fault_handler that runs on trap stack:

**Critical constraint: This function runs in trap context. It MUST NOT .await, yield, or call any async function. Only frame_alloc_sync() for allocation.**

```rust
pub enum FaultResult {
    Resolved,          // fault handled, return to user
    NeedsAsyncIO,      // file-backed page, needs async fetch (Phase 3)
    Error(FaultError),
}

pub enum FaultError {
    OutOfMemory,
    InvalidAccess,
    NotMapped,
}
```

`pub fn sync_fault_handler(vm_map: &mut VmMap, pmap: &mut Pmap, fault_va: VirtAddr, access_type: PageFaultAccessType) -> FaultResult`:

1. **Find VMA**: `vm_map.find_area(fault_va)` — if None, return Error(NotMapped)
2. **Check permissions**: if access_type not allowed by vma.prot, return Error(InvalidAccess)
3. **Classify fault**:

   a. **Anonymous fault (page not in any VmObject in chain)**:
      - Call `frame_alloc_sync()` — if None, return Error(OutOfMemory)
      - `pmap_zero_page(new_frame)` — zero the frame
      - Insert page into topmost VmObject: `vma.object.write().insert_page(offset, OwnedPage::anonymous(new_frame))`
      - `pmap_enter(fault_va_aligned, new_frame, vma.prot)` — install mapping
      - Return Resolved

   b. **COW fault (write to read-only page that VMA says is writable)**:
      - Walk shadow chain to find the page: `vma.object.read().lookup_page(offset)`
      - If page found and refcount of the Arc<RwLock<VmObject>> holding it is > 1 (shared):
        - `frame_alloc_sync()` for new frame
        - `pmap_copy_page(old_frame, new_frame)` — copy contents
        - Insert new frame into topmost VmObject
        - `pmap_enter(fault_va_aligned, new_frame, vma.prot | Write)` — map with write permission
        - Return Resolved
      - If page found and refcount == 1 (sole owner):
        - Just upgrade permissions: `pmap_protect(fault_va_aligned, vma.prot | Write)`
        - Return Resolved

   c. **File-backed fault (VmObject is Cached type)**:
      - Return NeedsAsyncIO — Phase 3 will handle this via fault_in_page().await

4. Page-align fault_va: `fault_va_aligned = fault_va & !0xFFF`
5. Compute object offset: `offset = (fault_va_aligned - vma.range.start) + vma.obj_offset`

Add `pub mod fault;` to os/src/mm/vm/mod.rs and re-export `sync_fault_handler` and result types.

**Integration point**: The existing trap handler in hal dispatches page faults. For now, just create the function — wiring it into the actual trap dispatch will happen when we integrate with the existing trap handler (which currently calls `handle_page_fault` on UserVmSpace).
