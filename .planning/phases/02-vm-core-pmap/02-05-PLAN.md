  <action>
Create host-runnable unit tests (cargo test --lib) for pure MI logic. These tests run on the host, not QEMU, following the Tier 1 pattern from Phase 1.

**1. os/src/mm/vm/tests.rs — VmObject and VmMap tests:**

Gate with `#[cfg(test)]` module.

VmObject tests:
- `test_vm_object_create`: create VmObject, verify size and empty pages
- `test_vm_object_insert_lookup`: insert page at offset, lookup returns it
- `test_vm_object_shadow_chain`: create parent, create shadow, verify shadow.backing points to parent, verify shadow_depth() == 1
- `test_vm_object_shadow_lookup_fallthrough`: insert page in parent, lookup from shadow finds it via chain walk
- `test_vm_object_shadow_override`: insert page in parent at offset X, insert different page in shadow at offset X, lookup from shadow returns shadow's page (not parent's)
- `test_vm_object_iterative_drop_deep_chain`: create a chain of 600 shadows (loop: shadow = new_shadow(shadow)), drop the outermost — must not stack overflow. Verify by checking that it completes without panic.
- `test_vm_object_iterative_drop_shared_stops`: create chain of 3, hold Arc to middle, drop outermost — middle and parent survive (Arc::try_unwrap fails on middle)

VmMap tests:
- `test_vma_monotonic_id`: create two VMAs, verify id2 > id1
- `test_vm_map_insert_find`: insert VMA, find_area returns it
- `test_vm_map_overlap_rejected`: insert two overlapping VMAs, second returns Err(Overlap)
- `test_vm_map_remove`: insert then remove, find_area returns None
- `test_vm_map_fork_creates_shadows`: insert VMA with VmObject, fork(), verify child VMA's object is a shadow of parent's object

Add `#[cfg(test)] mod tests;` to os/src/mm/vm/mod.rs.

**2. os/src/mm/allocator/tests.rs — Buddy allocator tests:**

Gate with `#[cfg(test)]` module.

- `test_buddy_alloc_free`: init with 1024 pages, alloc order-0, free, alloc again — same page returned (coalesced)
- `test_buddy_alloc_order`: alloc order-2 (4 pages), verify returned PPN is 4-aligned
- `test_buddy_coalesce`: alloc two adjacent order-0 blocks, free both, alloc order-1 — should succeed (buddies coalesced)
- `test_buddy_exhaustion`: alloc until None, verify available_pages() == 0
- `test_buddy_split`: alloc order-0 from a pool that only has order-10 blocks — verify splitting works

Add `#[cfg(test)] mod tests;` to os/src/mm/allocator/mod.rs.

**3. os/src/mm/pmap/tests.rs — ASID allocator tests (host-testable parts):**

- `test_asid_alloc_sequential`: alloc N ASIDs, verify all unique and sequential
- `test_asid_generation_rollover`: alloc MAX_ASID+1 ASIDs, verify generation incremented (mock the IPI barrier for host test — skip actual IPI, just verify generation logic)
- `test_asid_check_stale`: alloc ASID, force rollover, check_asid returns false

Add `#[cfg(test)] mod tests;` to os/src/mm/pmap/mod.rs.

For host testability, the ASID allocator's IPI barrier should be behind a trait or cfg(test) mock that skips actual IPI on host. Follow the Phase 1 pattern where `arch_irq` functions are cfg-gated for host.

**1. Integration test functions (add to main.rs after existing Phase 1 tests):**

`test_anonymous_page_fault()`:
- Create a new Pmap and VmMap
- Insert an anonymous VMA covering a test VA range (e.g., 0x1000_0000..0x1000_1000)
- Call sync_fault_handler with a read fault at 0x1000_0000
- Verify FaultResult::Resolved
- Call pmap_extract to verify the VA is now mapped
- Print `[PASS] anonymous page fault resolved`

`test_cow_fault()`:
- Create VmMap with anonymous VMA + VmObject containing a page
- Fork the VmMap (creates shadow chain)
- Call sync_fault_handler with a write fault on the child's mapping
- Verify a new frame was allocated (pmap_extract returns different PPN than parent)
- Print `[PASS] COW fault resolved`

`test_iterative_drop_500()`:
- Create a 500-deep shadow chain
- Drop it
- If we reach the next line without stack overflow: `[PASS] iterative drop 500-deep chain`

`test_frame_alloc_sync_trap_context()`:
- Call frame_alloc_sync() — verify it returns Some
- Call frame_free() on the result
- Call frame_alloc_sync() again — verify it returns Some
- Print `[PASS] frame_alloc_sync works in sync context`

`test_asid_rollover()` (if feasible on QEMU — may need to be simplified):
- Allocate ASIDs in a loop until rollover
- Verify TLB flush occurred (check generation incremented)
- Print `[PASS] ASID rollover with TLB flush`
