**3. Page cache** (`kernel/src/fs/page_cache.rs`):

Per CONTEXT.md: `HashMap<(VnodeId, u64), PageState>` with 3-state CAS.

```rust
use alloc::sync::Arc;
use alloc::vec::Vec;
use core::task::Waker;
use spin::Mutex;

use hal_common::PhysAddr;
use super::vnode::VnodeId;

/// Page cache entry states.
pub enum PageState {
    /// Page is being fetched — waiters will be woken when done.
    Fetching(Vec<Waker>),
    /// Page is cached in memory.
    Cached(PhysAddr),
}
```

The page cache is a global `Mutex<BTreeMap<(VnodeId, u64), PageState>>` (or HashMap with hashbrown).

Key operations:
- `page_cache_lookup(vnode_id, page_offset) -> PageCacheLookup` — returns `Hit(PhysAddr)`, `Pending` (caller should await), or `Miss` (caller should initiate fetch)
- `page_cache_start_fetch(vnode_id, page_offset)` — transitions Absent → Fetching, returns a handle
- `page_cache_complete_fetch(vnode_id, page_offset, phys)` — transitions Fetching → Cached, wakes all waiters
- `page_cache_wait(vnode_id, page_offset) -> impl Future<Output = PhysAddr>` — registers Waker on Fetching entry, resolves when Cached

The 3-state CAS pattern:
1. Caller checks cache: Miss → insert `Fetching(vec![])`, go fetch from vnode
2. Other callers check cache: see `Fetching` → register Waker, suspend
3. First caller completes fetch: insert `Cached(phys)`, wake all waiters

Use `Mutex` (not CAS on atomic) since the map itself needs locking. The "CAS" is conceptual — the state transitions are atomic under the lock.

**1. Filesystem syscalls** (`kernel/src/fs/syscalls.rs`):

Implement the six syscalls from VFS-05. All are async (they may need to go through the delegate).

a) **sys_open(path: &str, flags: u32) -> Result&lt;u32, Errno&gt;**:
   - Walk path using `path_lookup` from root vnode
   - Create `OpenFile { vnode, offset: AtomicU64::new(0), flags }`
   - Allocate fd in current task's FdTable
   - Return fd number

b) **sys_read(fd: u32, buf: &mut [u8]) -> Result&lt;usize, Errno&gt;**:
   - Look up fd in FdTable → get `Arc<OpenFile>`
   - Read current offset, calculate which page(s) to read
   - For each page: check page cache → if miss, fetch via vnode.read_page through delegate → copy data to user buffer via `uiomove`
   - Advance offset by bytes read
   - Return bytes read

c) **sys_write(fd: u32, buf: &[u8]) -> Result&lt;usize, Errno&gt;**:
   - Similar to read but writes through page cache
   - For Phase 3: write-through to delegate (mark page dirty, delegate writes back)
   - Advance offset by bytes written

d) **sys_close(fd: u32) -> Result&lt;(), Errno&gt;**:
   - Remove fd from FdTable
   - Arc<OpenFile> dropped when last reference gone

e) **sys_stat(path: &str) -> Result&lt;VnodeStat, Errno&gt;**:
   - Walk path, call vnode.stat()

f) **sys_readdir(fd: u32) -> Result&lt;Vec&lt;DirEntry&gt;, Errno&gt;**:
   - Look up fd, verify it's a directory vnode
   - Call vnode.readdir() through delegate

    access_type: PageFaultAccessType,

Logic per CONTEXT.md decisions:
1. Find VMA containing `fault_va`
2. Calculate `offset_in_vma = fault_va - vma.va_start`
3. If `offset_in_vma < vma.file_size` (file-backed region):
   a. Calculate file page offset: `file_page = (vma.file_offset + offset_in_vma) / PAGE_SIZE`
   b. Fetch page from page cache: `page_cache_lookup(vnode_id, file_page)`
      - If miss: initiate fetch via vnode.read_page, await completion
      - If fetching: await completion
      - If cached: use directly
   c. **Boundary page check**: if `offset_in_vma + PAGE_SIZE > vma.file_size`:
      - Allocate fresh frame via `frame_alloc().await`
      - Copy file portion from page cache frame
      - Zero the tail (from file_size to end of page)
      - Map this anonymous frame (not the page cache frame)
   d. **Normal file page**: map page cache frame read-only (MAP_PRIVATE COW)
      - Insert as `OwnedPage::new_cached(phys)` in VmObject
      - Map via pmap_enter with read-only permissions
      - Write fault on this page later triggers COW (handled by sync_fault_handler)
4. If `offset_in_vma >= vma.file_size` (zero-fill region beyond file):
   - Allocate zeroed frame, insert as anonymous, map with full VMA permissions
   - This is the BSS/zero-fill case for ELF segments
