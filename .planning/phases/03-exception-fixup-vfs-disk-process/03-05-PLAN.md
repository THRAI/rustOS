**3. exec() implementation**:

```rust
pub async fn exec(task: &Arc<Task>, path: &str, argv: &[&str]) -> Result<(), Errno> {
```

Steps:
1. Open the ELF file via VFS: resolve path to vnode, read ELF header (first page) through page cache or direct delegate read.
2. Validate ELF magic (`\x7fELF`), class (ELFCLASS64), machine (EM_RISCV = 243), type (ET_EXEC).
3. Parse program headers: iterate PT_LOAD segments.
4. **Reset vm_map**: Drop all existing VMAs and unmap from pmap. Create fresh VmMap.
5. For each PT_LOAD segment:
   - Calculate `va_start = p_vaddr` (page-aligned down), `va_end = p_vaddr + p_memsz` (page-aligned up)
   - Convert `p_flags` to `MapPerm` (PF_Râ†’R, PF_Wâ†’W, PF_Xâ†’X, always add U)
   - Create VmArea with:
     - `vnode = Some(Arc::clone(&elf_vnode))`
     - `file_offset = p_offset` (adjusted for page alignment)
     - `file_size = p_filesz` (adjusted for page alignment offset)
     - `area_type = VmAreaType::FileBacked`
     - Anonymous VmObject (pages will be faulted in on demand)
   - Insert VMA into new vm_map
6. Create stack VMA: anonymous, RW, at a conventional address (e.g., `0x7FFF_F000` top, 8 pages = 32KB)
7. Set up initial user stack: push argv strings, build argc/argv pointers per RISC-V ABI (a0=argc, a1=argv on stack, sp aligned to 16)
8. Set task's trap frame: `sepc = e_entry`, `sp = stack_top - argv_size`, `sstatus` with SPP=0 (user mode), SPIE=1
9. Replace task's vm_map and pmap with the new ones

**CRITICAL â€” demand paging**: exec() does NOT read any file data or allocate frames for PT_LOAD segments. It only creates VMAs with vnode references. The fault handler (fault_in_page from plan 03-04) handles actual page loading when the process first accesses each page.

The three fault paths per CONTEXT.md:
1. Pure file page (`offset_in_vma < file_size` and page-aligned within file): map page cache frame read-only, COW on write
2. Boundary page (`offset_in_vma < file_size` but page crosses file_size boundary): immediate anonymization â€” allocate frame, memcpy file portion, zero tail

Path 1 and 2 are handled by `fault_in_page().await` (plan 03-04). Path 3 is already handled by the existing anonymous fault path. The sync_fault_handler just needs to correctly classify and route.
