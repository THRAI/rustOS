## Objective

Replace the current `OpenFile`-based fd table with the BSD three-layer model (FdTable → FileDescription → FileObject), implement pipes with async waker integration, add static device nodes, and wire up dup/dup2/dup3/pipe2 syscalls. This is the foundation for signals (SIGPIPE) and the remaining syscall surface.

## Context

Current state:
- `fd_table.rs`: `FdTable` maps `u32 → Arc<OpenFile>` where `OpenFile` holds `vnode: Arc<dyn Vnode>`, `offset: AtomicU64`, `flags: OpenFlags`
- `vnode.rs`: `Vnode` trait with `vnode_id/vtype/size/path` methods, `Ext4Vnode` impl
- `user_task.rs`: Syscall dispatch handles READ/WRITE/OPENAT/CLOSE/FSTAT etc. WRITE to fd 1/2 goes to UART. READ goes through page cache.
- `fork.rs`: `fork()` calls `fd_table.fork()` which clones the BTreeMap of `Arc<OpenFile>`
- `exec.rs`: No CLOEXEC handling yet

User decisions (locked):
- Three-layer model: FdTable (per-process Vec<Option<(Arc<FileDescription>, FdFlags)>>) → FileDescription (Arc, shared across dup/fork, holds offset + flags + FileObject) → FileObject enum (Vnode | Pipe | Device)
- Pipe: Fixed 4KB ring buffer, IrqSafeSpinLock, Option<Waker> per end, PIPE_BUF=4096 atomic writes, register waker WHILE holding lock, wake AFTER releasing lock
- Devices: Static table, not full devfs. /dev/null (read=EOF, write=discard), /dev/zero (read=zeros, write=discard), /dev/console (SBI putchar write, ConsoleReadFuture with 10ms timer poll)
- dup/dup2: Clone Arc<FileDescription>, clear CLOEXEC. dup3: set CLOEXEC if O_CLOEXEC. pipe2(O_CLOEXEC): both fds get CLOEXEC.
- FdFlags: bitflags with CLOEXEC = 0o2000000
- exec: iterate entries, .take() all slots with CLOEXEC set

```rust
// Layer 1: FdTable (per-process)
pub struct FdTable {
    entries: Vec<Option<(Arc<FileDescription>, FdFlags)>>,
}

bitflags::bitflags! {
    pub struct FdFlags: u32 {
        const CLOEXEC = 0o2000000;
    }
}

// Layer 2: FileDescription (shared across dup/fork)
pub struct FileDescription {
    pub offset: AtomicU64,
    pub status_flags: AtomicU32,  // O_NONBLOCK etc
    pub file_object: FileObject,
}

// Layer 3: FileObject (the backing resource)
pub enum FileObject {
    Vnode(Arc<dyn Vnode>),
    Pipe(Arc<PipeEnd>),
    Device(DeviceNode),
}
```

FdTable methods:
- `insert(fd: FileDescription, flags: FdFlags) -> u32` — find lowest free slot (scan Vec for None), insert, return fd number
- `insert_at(fd_num: u32, desc: Arc<FileDescription>, flags: FdFlags) -> Result<(), Errno>` — for dup2
- `get(fd: u32) -> Option<&(Arc<FileDescription>, FdFlags)>`
- `remove(fd: u32) -> Option<Arc<FileDescription>>`
- `fork() -> FdTable` — clone all entries (Arc::clone on FileDescription)
- `exec_cloexec()` — iterate, .take() all slots where flags contains CLOEXEC
- `dup(old_fd: u32) -> Result<u32, Errno>` — clone Arc<FileDescription>, insert with empty FdFlags
- `dup2(old_fd: u32, new_fd: u32) -> Result<u32, Errno>` — close new_fd if open, insert_at with empty FdFlags
- `dup3(old_fd: u32, new_fd: u32, flags: FdFlags) -> Result<u32, Errno>` — like dup2 but set flags

Pre-populate fd 0/1/2 with /dev/console FileDescriptions in `FdTable::new_with_stdio()`.

<action>
Create `kernel/src/fs/pipe.rs`:

```rust
pub const PIPE_BUF: usize = 4096;
const RING_SIZE: usize = 4096;

struct PipeInner {
    buf: [u8; RING_SIZE],
    head: usize,       // read position
    tail: usize,       // write position
    reader_closed: bool,
    writer_closed: bool,
    reader_waker: Option<Waker>,
    writer_waker: Option<Waker>,
}

pub struct PipeEnd {
    inner: IrqSafeSpinLock<PipeInner>,
    is_read_end: bool,
}
```

Key behaviors per CONTEXT.md decisions:
- `fn used(&self) -> usize` = `(tail - head) % RING_SIZE` (or equivalent with wrapping)
- `fn free(&self) -> usize` = `RING_SIZE - 1 - used()` (one slot wasted to distinguish full/empty)
- **Read poll**: Lock inner. If `used() > 0`, copy min(buf.len(), used()) bytes from ring, advance head, register no waker, wake writer_waker AFTER dropping lock, return Ready(Ok(n)). If `used() == 0 && writer_closed`, return Ready(Ok(0)) (EOF). If `used() == 0`, store waker in reader_waker WHILE holding lock, return Pending.
- **Write poll**: Lock inner. If `reader_closed`, return Ready(Err(EPIPE)). If `data.len() <= PIPE_BUF`: all-or-nothing — if `free() < data.len()`, store waker, return Pending; else write all. If `data.len() > PIPE_BUF`: partial write allowed — write min(free(), data.len()). Wake reader_waker AFTER dropping lock.
- **Reader Drop**: Lock, set `reader_closed = true`, take writer_waker. Drop lock. Wake writer.
- **Writer Drop**: Lock, set `writer_closed = true`, take reader_waker. Drop lock. Wake reader.

`pub fn pipe_create() -> (Arc<PipeEnd>, Arc<PipeEnd>)` — creates shared PipeInner, returns (read_end, write_end).

Wire `PipeEnd` into `FileObject::Pipe(Arc<PipeEnd>)`.

Add `SYS_PIPE2` (nr 59) to `user_task.rs` syscall dispatch:
- a0 = user pointer to `[i32; 2]` for fd numbers, a1 = flags
- Call `pipe_create()`, wrap each end in `FileDescription`, insert into FdTable
- If flags & O_CLOEXEC, set FdFlags::CLOEXEC on both
- copyout the two fd numbers to user pointer

Pipe read/write in syscall dispatch: match `FileObject::Pipe(ref end)` → call `PipeReadFuture`/`PipeWriteFuture` which implement `Future` using the poll logic above.
</action>

Device operations (called from FileObject::Device match arms in syscall dispatch):

- **Null read**: return Ok(0) (EOF)
- **Null write**: return Ok(len) (discard all)
- **Zero read**: fill user buffer with zeros via uiomove from a pre-zeroed kernel page. Return Ok(len).
- **Zero write**: return Ok(len) (discard)
- **Console write**: SBI putchar loop (existing UART write logic, moved from hardcoded fd 1/2 check)
- **Console read**: `ConsoleReadFuture` — poll `sbi_console_getchar()`. If data available, copyout byte, return Ready. If no data, register 10ms timer wheel wakeup, return Pending. This is the async polling loop per CONTEXT.md (SBI has no input IRQ).

Static device table for path resolution:
```rust
pub fn lookup_device(name: &str) -> Option<DeviceNode> {
    match name {
        "null" => Some(DeviceNode::Null),
        "zero" => Some(DeviceNode::Zero),
        "console" => Some(DeviceNode::Console),
        _ => None,
    }
}
```

Update `path.rs` resolve: if path starts with `/dev/`, call `devfs::lookup_device()` and return a FileObject::Device instead of going through ext4.
