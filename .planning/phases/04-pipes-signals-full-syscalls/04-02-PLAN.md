## Objective

Implement POSIX signal delivery: sigaction, sigprocmask, kill, signal frame construction (sendsig), sigreturn, and the sigcode trampoline page. Add process groups for shell pipeline support. Wire SIGPIPE into the pipe write path from Plan 04-01.

## Context

Current state:
- `task.rs`: Task struct has pid, parent (Weak), children, vm_map, fd_table, state, exit_status, parent_waker, trap_frame, pmap, kstack, brk. No signal fields.
- `user_task.rs`: `run_tasks` loop: pmap_activate → trap_return → user_trap_handler → pmap_deactivate → yield. Syscall dispatch in `dispatch_syscall`. SyscallId::SIGACTION/SIGPROCMASK/SIGRETURN currently return 0 (stub).
- `exit_wait.rs`: sys_exit stores exit_status, sets zombie, wakes parent. WaitChildFuture polls children for zombie.
- `trap.S`: __user_trap saves user regs to TrapFrame, restores kernel callee-saved, returns to run_tasks.
- pcb_onfault: Working in both kernel_trap_handler and user_trap_handler for copy_user fixup.

User decisions (locked):
- Sigcode page: dedicated RO page at fixed VA, contains `ecall SYS_rt_sigreturn` trampoline. Signal frame's `ra` points here.
- Core 14 signals: SIGKILL, SIGTERM, SIGCHLD, SIGPIPE, SIGSEGV, SIGBUS, SIGFPE, SIGILL, SIGINT, SIGSTOP, SIGCONT, SIGALRM, SIGUSR1, SIGUSR2.
- sigaction flags: SA_SIGINFO, SA_RESTART, SA_NOCLDWAIT, SA_ONSTACK.
- Per-task signal mask: pending + blocked as u64 bitmaps in Task struct.
- sendsig: build sigframe in kernel memory, copyout to user stack. If user SP trashed → copyout fails via pcb_onfault → kill with SIGILL.
- sigaltstack: ss_sp + ss_size + ss_flags in Task. SA_ONSTACK + SIGSEGV → use altstack.
- sigreturn sanitization: (1) SPP cleared + SPIE set, (2) sepc < USER_MAX_VA, (3) only restore GPRs + sanitized sstatus/sepc from ucontext.
- Restart whitelist (SA_RESTART): read, write, wait4. Always EINTR: nanosleep, futex_wait.
- Async injection: kill does pending_signals.fetch_or(bit) + top_level_waker.wake(). Blocking polls check has_unmasked_pending_signals() at entry → Poll::Ready(Err(EINTR)).
- Process groups: pgid: u32 in Task, inherited from parent on fork. setpgid/getpgid syscalls. kill(-pgrp) iterates global task list.

**Sigcode page setup:**

Define `SIGCODE_VA: usize = 0x0000_003F_FFFE_0000` (one page below stack region). The page contains the sigreturn trampoline:
```asm
// At offset 0:
li a7, 139        // SYS_rt_sigreturn
ecall
```
Assemble these bytes as a const array. In exec (after vm_map reset), allocate a frame, write the trampoline bytes, map it RO+U at SIGCODE_VA via pmap_enter. Also map it in Task::new_init().

**Signal frame (sigframe) layout on user stack** (grows downward):
```rust
#[repr(C)]
struct SigFrame {
    // ucontext: saved registers for sigreturn to restore
    uc_mcontext: [usize; 32],  // x0-x31 (x0 always 0)
    uc_sepc: usize,
    uc_sstatus: usize,
    // siginfo (if SA_SIGINFO)
    si_signo: i32,
    si_errno: i32,
    si_code: i32,
    _pad: i32,
    si_pid: u64,    // sender pid
    si_addr: u64,   // fault address (for SIGSEGV/SIGBUS)
}
```

**sendsig(task, sig):**
1. Look up sig_actions[sig]. If SIG_IGN → return. If SIG_DFL → apply default action (Term → do_exit, Ignore → return, etc.).
2. Save current trap frame GPRs + sepc + sstatus into a kernel-side SigFrame.
3. Determine user SP: if SA_ONSTACK and sigaltstack configured and not already on altstack → use ss_sp + ss_size. Otherwise use current user sp from trap frame.
4. Compute frame_sp = (user_sp - sizeof(SigFrame)) & !0xF (16-byte aligned).
5. Copyout SigFrame to frame_sp using uiomove(CopyOut). If copyout fails (pcb_onfault returns EFAULT) → kill process with SIGILL (corrupted stack, unrecoverable).
6. Modify trap frame for handler entry:
   - `sepc = sa_handler`
   - `x[10] (a0) = sig` (signal number)
   - `x[11] (a1) = frame_sp + offset_of(si_signo)` (siginfo pointer, if SA_SIGINFO)
   - `x[12] (a2) = frame_sp` (ucontext pointer)
   - `x[1] (ra) = SIGCODE_VA` (return to sigreturn trampoline)
   - `x[2] (sp) = frame_sp`
7. Update blocked mask: `blocked |= sa_mask | sig_bit(sig)` (block the signal during handler + sa_mask).

**check_pending_signals(task) -> bool:**
Called in `run_tasks` loop after `user_trap_handler` returns `TrapResult::Continue`, before `yield_now`. Calls `task.dequeue_signal()`. If Some(sig), calls `sendsig(task, sig)`. Returns true if a signal was delivered (so run_tasks skips yield and goes straight back to trap_return to execute the handler).

**sys_sigreturn(task):**
1. Read SigFrame from user sp (current trap frame sp) via copyin.
2. Restore GPRs from uc_mcontext into trap frame.
3. Sanitize: `sstatus = (uc_sstatus & !SSTATUS_SPP) | SSTATUS_SPIE`. Reject if `uc_sepc >= USER_MAX_VA` → SIGSEGV.
4. Restore sepc from uc_sepc.
5. Restore blocked mask from pre-signal state (stored in sigframe or tracked in task).

**Syscall restart mechanism:**
In dispatch_syscall, after the syscall returns, if the return value is -EINTR and a signal was just delivered:
- Check if the signal's SA_RESTART is set AND the syscall is in the restart whitelist (READ, WRITE, WAIT4).
- If yes: `trap_frame.sepc -= 4` (rewind to ecall), clear a0 return value. The next trap_return re-executes the ecall.
- If no: leave a0 = -EINTR.

Store `last_syscall_id` in Task (or a local in run_tasks) so the restart check knows which syscall was interrupted.

**EINTR guard in blocking futures:**
Add to pipe read/write poll, WaitChildFuture poll, and (later) nanosleep/futex:
```rust
if current_task.has_unmasked_pending_signals() {
    return Poll::Ready(Err(Errno::EINTR));
}
```

**SIGPIPE wiring:**
In pipe.rs PipeWriter::poll_write, when `reader_closed` is detected, before returning EPIPE, call `current_task.post_signal(SIGPIPE)`.

**SIGCHLD wiring:**
In sys_exit / do_exit, after setting zombie state, post SIGCHLD to parent: `parent.post_signal(SIGCHLD)`.

**Top-level waker registration:**
In `run_tasks`, at the start of each loop iteration (or in UserTaskFuture::poll), store `cx.waker().clone()` into `task.top_level_waker`. This ensures kill's waker.wake() re-enqueues the task to the run queue.

Actually, since run_tasks is an async fn (not a manual Future), the waker is implicit. The signal injection path needs to wake the task's executor-level Runnable. Store the waker in the first poll of run_tasks by wrapping the inner loop in a manual Future, or simpler: use a `SignalWakeHelper` future that captures the waker on first poll and stores it in task.top_level_waker, then immediately returns Ready. Call it once at the top of run_tasks.
