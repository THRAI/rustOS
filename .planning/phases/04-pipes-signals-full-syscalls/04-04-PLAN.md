## Objective

Wire signal delivery into the syscall restart mechanism, connect SIGPIPE to pipe death, add EINTR guards to all blocking futures, post SIGCHLD on child exit, and validate everything with QEMU integration tests. This is the final plan that makes all Phase 4 components work together as a coherent system.

## Context

After plans 04-01, 04-02, 04-03:
- FD model is three-layer with pipes, devices, dup
- Signal infrastructure exists: sigaction, sigprocmask, kill, sendsig, sigreturn, check_pending_signals at userret
- Syscalls are real: mmap/munmap/mprotect, lseek/fstat, clock_gettime/nanosleep, futex
- What's missing: the cross-cutting concerns that connect these subsystems

User decisions (locked):
- Restart whitelist (SA_RESTART → sepc -= 4): read, write, wait4. These are idempotent.
- Always EINTR: nanosleep (must write remaining time), futex_wait (user atomic may have mutated).
- Blocking poll guard: every blocking poll checks `has_unmasked_pending_signals()` at entry → Poll::Ready(Err(EINTR)).
- SIGPIPE: reader Drop sets reader_closed + wakes writers. Writer poll sees reader_closed → EPIPE → syscall layer posts SIGPIPE.
- SIGCHLD: child exit posts SIGCHLD to parent. SA_NOCLDWAIT → auto-reap (don't create zombie).
- wait4: interruptible. Signal during wait4 → EINTR (or restart if SA_RESTART).

**1. EINTR guard pattern for all blocking futures:**

Every blocking future's `poll` method gets a signal check at the top:

```rust
// In PipeReadFuture::poll, PipeWriteFuture::poll, FutexWaitFuture::poll, WaitChildFuture::poll:
fn poll(self: Pin<&mut Self>, cx: &mut Context<'_>) -> Poll<Self::Output> {
    // Signal guard: check before blocking
    if self.task.has_unmasked_pending_signals() {
        return Poll::Ready(Err(Errno::EINTR));
    }
    // ... normal poll logic ...
}
```

For nanosleep, the NanosleepFuture must track elapsed time:
```rust
struct NanosleepFuture {
    task: Arc<Task>,
    deadline_ticks: u64,
    start_ticks: u64,
}

impl Future for NanosleepFuture {
    fn poll(...) -> Poll<Result<(), (Errno, u64)>> {
        if self.task.has_unmasked_pending_signals() {
            let elapsed = now_ticks() - self.start_ticks;
            let total = self.deadline_ticks - self.start_ticks;
            let remaining_ns = ticks_to_ns(total - elapsed);
            return Poll::Ready(Err((Errno::EINTR, remaining_ns)));
        }
        // ... check deadline, register timer waker ...
    }
}
```

The nanosleep syscall handler writes remaining time to user `rem` pointer on EINTR.

**2. SA_RESTART mechanism in dispatch_syscall:**

After `dispatch_syscall` returns, if the return value is -EINTR, check:
```rust
// In run_tasks loop, after dispatch_syscall returns:
if ret_is_eintr && signal_was_just_delivered {
    let sa = task.sig_actions.lock()[delivered_sig as usize];
    if sa.sa_flags & SA_RESTART != 0 && is_restartable_syscall(last_syscall_id) {
        // Rewind PC to re-execute ecall
        task.trap_frame.lock().sepc -= 4;
        // Clear return value (will be set fresh by re-executed syscall)
    }
    // else: leave a0 = -EINTR, return to userspace
}

fn is_restartable_syscall(id: SyscallId) -> bool {
    matches!(id, SyscallId::READ | SyscallId::WRITE | SyscallId::WAIT4)
}
```

Implementation detail: the syscall dispatch must track which syscall was executing and whether a signal was delivered during it. Add a `last_syscall: Option<SyscallId>` and `signal_delivered: bool` to the run_tasks loop state. `check_pending_signals` sets `signal_delivered = true` when it delivers a signal. After userret from signal handler and sigreturn, the restart check fires.

Actually, simpler approach per the CONTEXT decisions: the restart check happens in `check_pending_signals` itself. When building the sigframe, if the interrupted context was a syscall return with -EINTR in a0, and SA_RESTART is set, and the syscall is in the whitelist, then modify the saved trap frame in the sigframe: `sepc -= 4` and clear a0. When sigreturn restores this frame, the CPU re-executes the ecall.

**3. SIGPIPE integration:**

In `pipe.rs` PipeWriteFuture::poll, after detecting `reader_closed`:
```rust
if inner.reader_closed {
    drop(inner); // release lock before posting signal
    self.task.post_signal(SIGPIPE);
    return Poll::Ready(Err(Errno::EPIPE));
}
```

The WRITE syscall dispatch in user_task.rs already returns -EPIPE to userspace. The signal delivery happens at the next userret via check_pending_signals.

**4. SIGCHLD on child exit:**

In `exit_wait.rs` `sys_exit` (or `do_exit` in user_task.rs), after setting zombie state:
```rust
fn do_exit(task: &Arc<Task>, status: i32) {
    // ... existing: set exit_status, set_zombie ...

    // Post SIGCHLD to parent
    if let Some(parent) = task.parent.upgrade() {
        // Check SA_NOCLDWAIT
        let nocldwait = {
            let actions = parent.sig_actions.lock();
            actions[SIGCHLD as usize].sa_flags & SA_NOCLDWAIT != 0
        };
        if nocldwait {
            // Auto-reap: skip zombie state, drop task immediately
            // Remove from parent's children list
            parent.children.lock().retain(|c| c.pid != task.pid);
        } else {
            parent.post_signal(SIGCHLD);
        }

        // Wake parent's waiter_waker (existing logic)
        if let Some(waker) = parent.parent_waker.lock().take() {
            waker.wake();
        }
    }
}
```

**5. Top-level waker registration:**

In `run_tasks`, at the start of each loop iteration, register the current executor waker as the task's top_level_waker:
```rust
async fn run_tasks(task: Arc<Task>) {
    loop {
        // Register top-level waker for signal injection
        // Use poll_fn to capture the waker
        core::future::poll_fn(|cx| {
            *task.top_level_waker.lock() = Some(cx.waker().clone());
            Poll::Ready(())
        }).await;

        // ... pmap_activate, trap_return, dispatch ...
    }
}
```

This ensures `kill()` → `post_signal()` → `top_level_waker.wake()` re-enqueues the task to the run queue even if it's parked in a pipe/futex/sleep wait.
