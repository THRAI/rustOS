  <action>
Create kernel/src/mm/vm/collapse.rs with tree_swap function implementing BSD vm_object_collapse logic:

1. Compare page counts: |child.pages| vs |parent.pages|
2. If |child| <= |parent|: migrate child pages to parent, update pindex offsets
3. If |parent| < |child|: migrate parent pages to child
4. Use min(|C|,|P|) insertions - never iterate the larger set
5. For each migrated page: acquire exBusy, update page.object pointer, release exBusy
6. Handle offset adjustments for shadow chain semantics (child offset != parent offset)

Key insight: BSD vm_object_collapse chooses migration direction based on density to minimize work.

Export: pub fn tree_swap(child: &mut VmObject, parent: &mut VmObject, offset: u64) -> Result<(), Errno>
