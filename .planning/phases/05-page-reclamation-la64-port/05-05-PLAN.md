<task type="auto">
  <name>Task 1: LA64 PTE encoding with negative permission logic</name>
  <files>kernel/src/mm/pmap/la64_pte.rs</files>
  <action>
Create kernel/src/mm/pmap/la64_pte.rs with LA64 PTE format:

LA64 PTE bits (48-bit PA, negative permissions):
- [47:12] PPN (physical page number)
- [11] NX (no execute - 1 means NOT executable)
- [10] NR (no read - 1 means NOT readable)
- [9] NW (no write - 1 means NOT writable)
- [8] PLV (privilege level - 0=kernel, 3=user)
- [7] MAT (memory access type - 0=coherent)
- [6] G (global)
- [5] D (dirty)
- [4] V (valid)

Key insight: LA64 uses NEGATIVE logic. Default (all zeros except V) is RWX. To restrict, set NR/NW/NX bits.

Implement:
1. encode_pte(pa: PhysAddr, perm: MapPerm) -> u64
   - Start with PPN from pa
   - Set V=1 (valid)
   - If !perm.READ: set NR=1
   - If !perm.WRITE: set NW=1
   - If !perm.EXEC: set NX=1
   - If perm.USER: set PLV=3, else PLV=0
2. decode_pte(pte: u64) -> (PhysAddr, MapPerm)
   - Extract PPN -> PhysAddr
   - Check NR/NW/NX bits, invert to get MapPerm
  </action>

  </verify>
  <done>la64_pte.rs exists with encode/decode functions, unit tests pass</done>
</task>
