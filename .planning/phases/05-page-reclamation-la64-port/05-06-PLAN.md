Create kernel/src/hal/la64/asid.rs (adapted from rv64 asid.rs):

1. LA64 ASID is 10-bit (max 1024), stored in ASID CSR
2. AsidAllocator with next_asid: AtomicU32, generation: AtomicU32
3. alloc() method:
   - Fetch-add next_asid
   - If next_asid >= 1024: call rollover()
   - Return (asid, generation)
4. rollover() method:
   - Increment generation
   - Reset next_asid to 1 (ASID 0 reserved for kernel)
   - Send IPI to all CPUs for TLB flush barrier
   - Wait for all CPUs to ack (synchronous barrier)

Key difference from rv64: 1024 ASIDs vs 65536, so rollover happens ~64x more frequently. IPI barrier is critical for correctness.

In kernel/src/hal/la64/mod.rs:

Create kernel/src/hal/la64/copy_user.S (mechanical translation of rv64 copy_user.S):

LA64 differences from rv64:
- Load/store: ld.d/st.d instead of ld/sd
- CSR access: csrrd/csrwr instead of csrr/csrw
- Register names: $r0-$r31 instead of x0-x31
- CRMD CSR instead of sstatus (bit 8 is PLV, bit 9 is IE)

Assembly structure:
1. Save landing pad address to pcb_onfault: la.global $t0, .Lfixup; st.d $t0, pcb_onfault_offset($tp)
2. Enable user memory access: csrrd $t0, CRMD; ori $t0, $t0, (1<<8); csrwr $t0, CRMD
3. Copy loop: ld.d from user VA, st.d to kernel VA, decrement count
4. Disable user access: csrrd $t0, CRMD; andi $t0, $t0, ~(1<<8); csrwr $t0, CRMD
5. Clear pcb_onfault: st.d $zero, pcb_onfault_offset($tp)
6. Return success: li $a0, 0; ret
7. .Lfixup landing pad: disable user access, return error: li $a0, -1; ret

In kernel/src/hal/la64/mod.rs:
