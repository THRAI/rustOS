# 错误处理统一修复清单

> **目标**：在全局分工之前，统一内核所有模块的错误类型，规范如下：
> 1. 在 `hal_common/errno.rs` 中增加 `pub type Result<T> = core::result::Result<T, Errno>;`
> 2. 在各模块 `mod.rs` 中做 re-export，让其它文件不再需要写完整路径
> 3. 将所有 `Result<T, i32>` / `Result<T, VmError>` / `Result<T, ()>` 统一改为 `Result<T, Errno>`

---

## 总览

| 优先级 | 模块 | 问题类型 | 影响范围 |
|--------|------|---------|---------|
| P0（先做） | `hal_common/errno.rs` | 缺少 `Result<T>` alias | 全局基础 |
| P0（先做） | `hal_common/mod.rs` | 未 re-export `KernelResult` | 全局基础 |
| P1 | `fs/ext4.rs` | 所有函数返回 `Result<T, i32>` | fs 子系统 |
| P1 | `fs/delegate.rs` | 所有 `ReplySlot<Result<X, i32>>` | fs 子系统 |
| P1 | `syscall/fs.rs` | `map_delegate_errno` 是局部私有函数 | fs syscall |
| P2 | `mm/vm/map/mod.rs` | `VmError` 无 `From<VmError> for Errno` | mm/vm |
| P2 | `mm/vm/vm_map.rs` | 存在第二个 `VmError` 定义 | mm/vm |
| P2 | `mm/vm/vm_object.rs` | `fetch_page_async` 用 `Errno`，其它地方可能用 `()` | mm/vm |
| P3 | `mm/uio.rs` | `EFAULT_RAW = 14` 魔法常量 | mm |
| P3 | `mm/mod.rs` | 无 re-export | mm |
| P3 | `fs/mod.rs` | 无 re-export | fs |
| P3 | `proc/mod.rs` | 无 re-export | proc |
| P3 | `syscall/mod.rs` | 无 re-export | syscall |

## Step 1：在 `hal_common/errno.rs` 增加 `Result<T>` alias

**文件**：`kernel/src/hal_common/errno.rs`

**当前状态**：只有 `Errno` 枚举定义，没有 `Result` alias。

**修改**：在文件末尾（`impl Errno` 块之后）追加：

```rust
/// Kernel-wide result type, error is always an errno code.
pub type KernelResult<T> = core::result::Result<T, Errno>;
```

> 为什么用 `KernelResult` 而非 `Result`？`no_std` 环境下 `core::result::Result` 已被 prelude 引入，如果直接命名 `Result` 会与 `core::result::Result` 产生歧义（需在每个作用域用 `use`）。使用 `KernelResult` 更明确。若组长坚持用 `Result`，则在每个文件顶部加 `use crate::hal_common::Result;` 来遮蔽 prelude。

---

## Step 2：在 `hal_common/mod.rs` re-export

**文件**：`kernel/src/hal_common/mod.rs`

**当前状态**（已有的 re-export）：
```rust
pub use errno::Errno;
// ... 其它 pub use
```

**修改**：增加一行：
```rust
pub use errno::KernelResult;
```

完成后其它模块只需 `use crate::hal_common::KernelResult;` 即可。

## Step 3：`fs/ext4.rs` — 替换所有 `Result<T, i32>` 为 `Result<T, Errno>`

**文件**：`kernel/src/fs/ext4.rs`

**当前问题**：所有 15+ 个公开函数都是 `Result<..., i32>`，返回的是
`lwext4` C 库的原始错误码（正数，如 `9` = EBADF），有时经过 `-(rc.abs())` 变成负数。

**例子（当前）**：
```rust
// ext4.rs line 66
pub fn mount() -> Result<(), i32> { ... }

// ext4.rs line 136
return Err(-(rc.abs()));    // 返回 -22 表示 EINVAL
```

**修改方案**：

1. 在文件顶部引入 `Errno`：
   ```rust
   use crate::hal_common::Errno;
   ```

2. 增加内部转换函数（仅在此文件使用）：
   ```rust
   /// 将 lwext4 返回的正整数错误码映射为 Errno。
   fn lwext4_err(rc: i32) -> Errno {
       match rc.unsigned_abs() {
           1 => Errno::Eperm,
           2 => Errno::Enoent,
           5 => Errno::Eio,
           9 => Errno::Ebadf,
           12 => Errno::Enomem,
           17 => Errno::Eexist,
           19 => Errno::Enodev,
           20 => Errno::Enotdir,
           21 => Errno::Eisdir,
           22 => Errno::Einval,
           24 => Errno::Emfile,
           39 => Errno::Enotempty,
           _  => Errno::Eio,
       }
   }
   ```

3. 将所有函数签名从 `Result<T, i32>` → `Result<T, Errno>`，并将
   `return Err(-(rc.abs()))` 替换为 `return Err(lwext4_err(rc))`。

**涉及函数（共 15 个）**：
- `mount()` — line 66
- `open()` — line 80
- `read()` — line 92
- `write()` — line 97
- `truncate()` — line 102
- `close()` — line 111
- `stat()` — line 122
- `dir_open()` — line 164
- `dir_close()` — line 190
- `mkdir()` — line 199
- `unlink()` — line 206
- `link()` — line 213
- `rename()` — line 220
- `symlink()` — line 227
- `readlink()` — line 234
- `cache_flush()` — line 240

## Step 4：`fs/delegate.rs` — 替换所有 `ReplySlot<Result<X, i32>>`

**文件**：`kernel/src/fs/delegate.rs`

**当前问题**：所有请求的 reply 槽都是 `Result<X, i32>`（共 16+ 个变体），
即 ext4.rs 的 i32 错误码直接穿透到这里。

**例子（当前）**：
```rust
// delegate.rs line 52
reply: ReplySlot<Result<FsFileHandle, i32>>,
// line 58
reply: ReplySlot<Result<usize, i32>>,
// line 68
reply: ReplySlot<Result<(), i32>>,
// ... 以此类推 16+ 行
```

**修改方案**：

1. 引入 `Errno`（如果未 import）：
   ```rust
   use crate::hal_common::Errno;
   ```

2. 全局替换所有 `ReplySlot<Result<X, i32>>` → `ReplySlot<Result<X, Errno>>`。

3. `delegate` loop 内部处理 ext4 函数返回值时，用 `ext4::lwext4_err()` 转换（或让 ext4.rs 函数完成 Step 3 后自然返回 `Errno`，此时无需额外转换）。

**建议实施顺序**：先完成 Step 3（ext4.rs），delegate.rs 就只需改类型签名，不需改逻辑。

---

## Step 5：`syscall/fs.rs` — 将 `map_delegate_errno` 移至 `fs/delegate.rs`

**文件**：`kernel/src/syscall/fs.rs`

**当前问题**：`map_delegate_errno(code: i32) -> Errno`（line 246）是 `syscall/fs.rs` 的私有函数，但 delegate/ext4 更改后将不再需要它；同时它目前是唯一的 i32→Errno 映射，理应在 fs 内部而非 syscall 层。

**完成 Step 3+4 后**：`map_delegate_errno` 调用处变为不再需要（delegate 已返回 `Errno`），可以直接删除该函数，并将各处 `.map_err(map_delegate_errno)?` 改为 `.map_err(|e| e)?`（即 `?` 直接传播）。

---

## Step 6：`mm/vm/map/mod.rs` 和 `mm/vm/vm_map.rs` — 为 `VmError` 实现 `From<VmError> for Errno`

**文件**：
- `kernel/src/mm/vm/map/mod.rs`（line 14 定义了 `VmError`）
- `kernel/src/mm/vm/vm_map.rs`（line 87 存在**第二个** `VmError` 定义）

**当前问题 1**：`VmError` 有两套定义，且都没有 `From<VmError> for Errno` 的 impl，
需要在 syscall boundary 写 `.map_err(|_| Errno::Einval)` 手动转换。

**当前 VmError 变体**：
```rust
// map/mod.rs
pub enum VmError {
    Overlap,
    NotFound,
    InvalidRange,
}

// vm_map.rs
pub enum VmError {
    Overlap,
    NotFound,
    InvalidRange,
    // ...可能有更多
}
```

**修改方案**：

1. **合并两个 VmError**：只保留 `mm/vm/map/mod.rs` 里的定义，在 `vm_map.rs` 中 `use super::map::VmError;`（或通过 `mm/vm/mod.rs` re-export）。

2. **增加 `From` impl**（放在 `map/mod.rs` 末尾）：
   ```rust
   use crate::hal_common::Errno;

   impl From<VmError> for Errno {
       fn from(e: VmError) -> Errno {
           match e {
               VmError::Overlap      => Errno::Einval,
               VmError::NotFound     => Errno::Einval,
               VmError::InvalidRange => Errno::Einval,
           }
       }
   }
   ```

3. **完成后**：调用 vm 函数的地方可以直接用 `?` 传播，不需再写 `.map_err(|_| Errno::Einval)`。

---

## Step 7：`mm/vm/vm_object.rs` — 统一返回类型

**文件**：`kernel/src/mm/vm/vm_object.rs`

**当前问题**：
- `fetch_page_async()` 返回 `Result<Arc<VmPage>, Errno>`（正确）
- `grab_for_fault()` 返回值类型需检查——如果是 `Result<Arc<VmPage>, ()>`，需改为 `Result<Arc<VmPage>, Errno>`

**修改方案**：
```rust
// 将 Err(()) 替换为合适的 Errno，例如：
Err(())  →  Err(Errno::Enomem)   // 物理内存分配失败
Err(())  →  Err(Errno::Eio)      // 后备存储读取失败
```

具体需要把 `grab_for_fault` 的签名从 `..., ()>` 改为 `..., Errno>`，并在各 `Err(())` 处替换为语义正确的 `Errno`。

---

## Step 8：`mm/uio.rs` — 消除魔法常量 `EFAULT_RAW`

**文件**：`kernel/src/mm/uio.rs`

**当前问题**（line 26）：
```rust
const EFAULT_RAW: usize = 14;  // 魔法常量
```
并在 line 81 使用：
```rust
if ret == EFAULT_RAW { ... }
```

**修改方案**：删除常量，直接用：
```rust
if ret == Errno::Efault.as_i32() as usize { ... }
```

或者根据函数语义，将 raw usize 的 `ret` 直接转换为 `Errno` 后比对，消除反射比较。

## Step 9：各模块 `mod.rs` 中的 re-export

完成上述修改后，在各 `mod.rs` 中加入 re-export，让其他文件可以用短路径：

### `kernel/src/hal_common/mod.rs`
```rust
// 已有
pub use errno::Errno;
// 新增
pub use errno::KernelResult;
```

### `kernel/src/mm/mod.rs`
```rust
// 新增
pub use uio::{UioResult, uiomove_from_user, uiomove_to_user};
pub use vm::map::VmError;
```

### `kernel/src/mm/vm/mod.rs`
```rust
// 新增
pub use map::VmError;
pub use vm_map::VmMap;
pub use vm_object::{VmObject, VmPage};
pub use fault::{FaultResult, FaultError, resolve_fault};
```

### `kernel/src/fs/mod.rs`
```rust
// 新增
pub use delegate::{fs_open, fs_read, fs_write /*...*/};  // 视需求决定
pub use fd_table::FdTable;
pub use vnode::Vnode;   // 已有
```

### `kernel/src/proc/mod.rs`
```rust
// 新增
pub use task::Task;
pub use pid::Pid;
pub use syscall_result::SyscallResult;
```

### `kernel/src/syscall/mod.rs`
```rust
// 新增
pub use self::fs::*;       // 或者按需选择
```

> **注意**：`syscall` 层的 re-export 谨慎处理，避免将内部 helper（如以前的 `map_delegate_errno`）暴露出去。

## 实施顺序建议

```
Step 1 → Step 2   （基础类型，不涉及任何逻辑变更）
     ↓
Step 3 → Step 4 → Step 5   （fs 子系统，一个人搞定，改完后 CI 跑 fs 相关测试）
     ↓
Step 6 → Step 7   （mm/vm，另一个人搞定）
     ↓
Step 8             （mm/uio，简单改一行）
     ↓
Step 9             （全部 mod.rs re-export，最后补上，减少冲突）
```

并行方案（两人分工）：
- **人员 A**：Step 1 + Step 2 + Step 3 + Step 4 + Step 5 + Step 9（fs 部分）
- **人员 B**：等 Step 1+2 合并后，做 Step 6 + Step 7 + Step 8 + Step 9（mm 部分）

## 验证方法

每完成一个 Step，跑：
```bash
cargo build -p kernel 2>&1 | grep -E "error\[|error:" | head -30
```

全部完成后，重点检查以下两点：
1. `grep -rn "Result<.*i32" kernel/src/` 应该只剩 `ext4.rs` 内部的 `lwext4_err` 辅助函数（如果留着的话），其它应为 0。
2. `grep -rn "Err(())" kernel/src/` 应返回 0 或只剩测试代码。
